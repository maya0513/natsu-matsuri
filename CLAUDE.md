# ブラウザゲームのプロジェクト

日本の夏祭りを体験できるwebアプリ

- 場所は神社周辺
- 時間はずっと夜
- 浴衣姿のレイヤーを操作して、屋台を回ってものを買ったり遊んだり、花火（一定周期で上がる）を見たりできる

## 描画方針

- 2.5D: three.js の 3D 空間に 2D スプライト（板ポリ/ビルボード）を立てる
- キャラ・屋台はスプライト、花火・提灯の光は 3D（パーティクル/ポイントライト）で演出

## tech stack

- typescript
- node
- pnpm
- vite plus(vite, vitest, oxlint, oxfmt, tsdown梱包)
- playwright（スモークE2Eのみ）
- three.js（ゲーム描画本体）
- howler.js（音: 祭囃子・花火・SE）
- preact(signals)（HUD・売買ダイアログ・持ち物画面などの UI 層のみ）
- tailwind v4（UI 層のスタイリング）
- just
- cloudflare Pages

## アセット方針

- 全アセットをコード生成（プロシージャル）。外部素材・手描き素材は使わない
- 画像: TS スクリプトでドット絵スプライトシート（PNG）を生成。キャラは 32px 基準
- 音: Web Audio で SE・BGM を合成（祭囃子はチップチューン風）
- アートスタイル: 夜の闇 + 発光表現（提灯・花火）に全振りしたレトロドット絵調
- 生成スクリプトもリポジトリ管理し、全アセットを再生成可能に保つ

## 設計方針

- ゲーム状態はプレーンなオブジェクト、更新は純粋関数（FP志向）+ 固定タイムステップのループ
- signals は「ゲーム状態 → UI」の橋渡し専用。rAF ループ内で signal を毎フレーム更新しない
- ゲームロジック（所持金・売買・花火の周期など）は Vitest で TDD、Playwright はスモークのみ
